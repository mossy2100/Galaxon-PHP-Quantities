## Built-in Quantity Types

The package includes built-in support for most quantity types in common usage and all with dedicated units. Each of the following quantity types has a dedicated class, which is a subclass of the central Quantity class. Most only provide default units and definitions for that particular quantity type, but several provide additional functionality or override the base Quantity class behaviour, and can be used as examples of how to develop your own quantity type classes.

There are many other quantity types for which no dedicated subclass is provided, as doing so would unnecessarily bloat the package.

Examples of quantity types without dedicated classes include momentum, torque, viscosity, electric field strength, heat capacity. Many more can be found at the Wikipedia article on [SI derived units](https://en.wikipedia.org/wiki/SI_derived_unit#By_field_of_application).

Only two Quantity type classes (Acceleration and Density) are included that add no functionality, as they have no dedicated units or conversions. However, they are quite common. These can be used as examples of how to add your own custom quantity types.

Using a dedicated subclass for a quantity type improves code readability and type safety, and can be a form of future-proofing in case additional functionality needs to be added later.  (See: [[Adding new quantity types]]).

## Creating new Quantities

There are two ways to create a new Quantity object:

### new *QuantityTypeSubclass*() (e.g. new Angle())

Use `new` to create a new object of the desired quantity type, e.g. `new Angle(45, 'deg')`, `new Density(1200, 'kg/m3')`, etc.

The unit is analysed to discover the quantity type. If the class doesn't match the unit, an exception will be thrown.

For example, `$x = new Time(100, 'kg')` will throw an exception because 'kg' is not a time unit.

This method for creating new quantities isn't technically required, since `Quantity::create()` is designed to handle all cases, but it does create more readable code.

You can also call `new Quantity()`, but this is generally not advised because it will throw an exception if a dedicated subclass exists for the relevant quantity type. Even if the class doesn't exist at the time the code is written, it could be added later.

### `Quantity::create()`

This will analyse the provided unit to determine the quantity type, and create an object of the correct class. If no class exists that matches the unit, a base Quantity object will be created. This approach alleviates the need to create quantity type classes.
