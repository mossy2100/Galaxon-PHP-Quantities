# Overview

Dimension codes represent the fundamental physical dimensions of a quantity using single-letter codes with optional exponents. Every quantity type has a unique dimension. For example:

- `L` - length
- `M` - mass
- `T` - time
- `MLT-2` - force (mass × length × time⁻²)

The standard dimension codes, as defined by the [International System of Quantities](https://en.wikipedia.org/wiki/International_System_of_Quantities) (ISQ), are as follows:

| Code | Name                | SI Base Unit |
| ---- | ------------------- | ------------ |
| `L`  | length              | m            |
| `M`  | mass                | kg           |
| `T`  | time                | s            |
| `I`  | electric current    | A            |
| `Θ`  | temperature         | K            |
| `N`  | amount of substance | mol          |
| `J`  | luminous intensity  | cd           |

A code of `1` indicates a dimensionless quantity, for example a concentration expressed in parts per million (`ppm`) or an interest rate expressed as a percentage (`%`).

The Quantities package tweaks and expands this system as follows:

1. The code `H` is used for temperature instead of `Θ` (theta), for easier typing, by myself and others. `H` was chosen for the word "heat", and also because capital theta has a little 'H' inside it.
2. The code `A` has been added for Angle quantities. In ISQ, an angle is viewed as dimensionless (code `1`) because an angle in radians is equal to the ratio of arc length to radius (i.e. L/L = 1). However, for the purpose of this package it was found useful to have a dedicated dimension for angles.
3. The code `D` has been added to support data quantities.
4. The code `C` has been added to support currency quantities (note, however, that "currency" refers to the *unit* type, and "money" refers to the *quantity* type).
5. Rather than a code of `1` to indicate a dimensionless quantity, the package uses an empty string.

The base units within the package have been similarly expanded.

1. Radians (`rad`) are an SI derived unit. Here, they are used as the SI base unit for angles.
2. Bytes (`B`) are used as the common base unit for data quantities. Although technically a bit (`b`) is more fundamental, in practical use bytes are much more common.
3. Troy ounces of gold (`XAU`) are used as the common base currency; any national currency would be biased.

Furthermore, as English units are relevant to certain package functions, every dimension is also assigned a base English unit. The concept of "English" units here encompasses both imperial and US customary units.

The resulting dimension codes and base units used within the package are thus:

| Code | Name                | SI base unit | English base unit |
| ---- | ------------------- | ------------ | ----------------- |
| L    | length              | m            | ft                |
| M    | mass                | kg           | lb                |
| T    | time                | s            | s                 |
| I    | electric current    | A            | A                 |
| H    | temperature         | K            | °R                |
| N    | amount of substance | mol          | mol               |
| J    | luminous intensity  | cd           | cd                |
| A    | angle               | rad          | °                 |
| D    | data                | B            | B                 |
| C    | currency            | XAU          | XAU               |

Thus, whenever converting a Quantity to SI base or English base units, the resulting derived unit will be a combination of units from the respective set.
