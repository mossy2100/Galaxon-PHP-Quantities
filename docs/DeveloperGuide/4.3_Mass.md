# Mass

The `Mass` class extends `Quantity` with convenience methods for configuring imperial or US customary part decomposition.

## Choosing a Parts System

Mass has no default part unit symbols because the choice between imperial and US customary units depends on context. Two convenience methods configure the parts system:

### Imperial Parts

```php
use Galaxon\Quantities\QuantityType\Mass;

Mass::setImperialParts();
// Part units: long ton, stone, pound, ounce
// Result unit: pound
```

The imperial system uses long tons (2240 lb) and stones (14 lb), but not grains:

```php
$weight = new Mass(157, 'lb');
echo $weight->formatParts();  // 11st 3lb

$cargo = new Mass(5000, 'lb');
echo $cargo->formatParts();   // 2LT 37st 2lb
```

### US Customary Parts

```php
Mass::setUsCustomaryParts();
// Part units: short ton, pound, ounce, grain
// Result unit: pound
```

The US customary system uses short tons (2000 lb) and grains (1/7000 lb), but not stones:

```php
$weight = new Mass(157, 'lb');
echo $weight->formatParts();  // 157lb

$cargo = new Mass(5000, 'lb');
echo $cargo->formatParts();   // 2tn 1000lb
```

These methods also call `UnitService::loadSystem()` to ensure the relevant units are available.

## SI base unit

Note that the SI base unit for mass is the kilogram (`kg`), not the gram. The gram is defined with `all metric` prefixes, so `kg`, `mg`, `μg`, etc. are all available:

```php
$mass = new Mass(1500, 'g');
echo $mass->toSi();  // 1.5 kg
```
